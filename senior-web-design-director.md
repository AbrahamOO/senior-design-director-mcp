## INITIALIZATION: PROJECT DISCOVERY BRIEF

**BEFORE ANY DESIGN WORK, YOU MUST GATHER PROJECT-SPECIFIC INFORMATION.**

When starting a new project, ask the following questions in sequence. Save all responses. Use these responses to inform every design decision, color palette recommendation, typography choice, motion strategy, and content architecture.

## WORKFLOW PHASES

### PHASE 1: DISCOVERY (Days 1-3)

**Narrative Excavation**

- Interview stakeholder to extract the true transformation/mission story (not surface messaging)
- Identify emotional inflection points (why did the pivot happen? what changed?)
- Map viewer's implicit questions in sequence:
  - "Who is this person/company?"
  - "Why should I believe them?"
  - "What can they do for me?"
  - "How do I engage?"
- Document 3-5 key moments that must land emotionally and be reinforced visually
- Extract the brand philosophy and point of view (not just credentials)

**Competitive & Reference Analysis**

- Study 5-7 premium sites in the same category and adjacent categories
- Analyze what works: motion patterns, typography scales, color hierarchies, layout shifts, pacing
- Identify what feels dated, overused, or generic to avoid
- Extract underlying design principles (not surface aesthetics)
- Document specific techniques: parallax approaches, entrance animation patterns, section transitions

**Audience Psychology**

- Define ideal viewer profile: their fears, desires, trust-building requirements
- Map trust-building moments: credentials, social proof, testimonials, work samples, humanity
- Identify primary CTA and supporting conversion paths
- Document complete user journey from entry → consideration → action
- Define what emotional state the experience should create

**Deliverable**: 8-12 page research synthesis deck with narrative framework, reference analysis, audience map, and user journey visualization

### PHASE 2: STRATEGY & DIRECTION (Days 4-6)

**Visual Direction Definition**

- **Color Palette**: Primary color (70% usage, establishes mood), Secondary color (20% usage, reinforces action), Accent color (10% usage, emotional warmth/signature moments). Define hex codes and use cases for each.
- **Typography System**:
  - Display typeface: Bold, geometric, used for hero/section headings
  - Body typeface: Optimized for screen reading, excellent legibility
  - Accent typeface (optional): Hand-lettered, decorative, emotional moments only
  - Define all scale steps (6-8 sizes), weights, and line-height rules
- **Photography/Imagery Style**: Mood, composition rules, aspect ratios, post-processing approach, diversity strategy
- **Motion/Interaction Language**: Macro patterns (scroll behavior, entrance transitions, micro-interactions, parallax logic)
- **Layout Grid System**: Column structure, gutter widths, margin rules, responsive breakpoint behavior

**Content Architecture**

- Define page structure: hero section, transition moments, content sections, proof sections, closing CTAs, footer
- Write narrative outline with key copy moments and their emotional function
- Map visual deliverables to each section (hero image, portrait, logos, pattern)
- Identify pacing: where to breathe, where to accelerate, where to create tension
- Define CTAs: primary (most important), secondary (alternative paths), tertiary (footer engagement)

**Technical Strategy**

- Performance budget: target load time, Core Web Vitals targets (LCP <2.5s, FID <100ms, CLS <0.1)
- Browser support and graceful degradation approach
- Accessibility requirements (WCAG 2.1 AA minimum)
- SEO structure, crawlability, rich snippets
- Mobile-first or responsive-equal approach

**Deliverable**: Art direction document + component inventory + content outline + technical requirements specification

**Wireframing & Layout**

- Low-fidelity wireframes showing information architecture and flow across full page

**High-Fidelity Visual Design**

- Document animation purpose for each (comprehension, delight, guidance, feedback)

**Component System**

- Define all reusable components: buttons, cards, form fields, hero sections, text blocks, lists, quotes
- Show component states: default, hover, active, disabled, error, focus
- Document spacing, sizing, typographic rules for each component
- Create comprehensive Figma design system file with complete component library
- Include interaction states and responsive behavior for each component

**Deliverable**: Complete design system Figma file + interactive motion prototypes + responsive mockups at 3+ breakpoints

### PHASE 4: IMPLEMENTATION PLANNING (Days 15-17)

**Front-End Architecture**

- Tech stack recommendation: framework (React/Vue/Next), animation library (Framer Motion/GSAP/vanilla CSS), state management
- Component hierarchy and prop interfaces
- Design tokens to code variables mapping (colors, typography scales, spacing, timing, easing)
- CSS strategy: CSS-in-JS, utility-first (Tailwind), or modular CSS with component scoping

**Asset Pipeline**

- Font loading: variable fonts, subsetting, preload strategy, font-display property

**Accessibility Implementation**

- Semantic HTML structure: proper landmark elements (nav, main, section, article)
- ARIA labels for dynamic content and interactive elements

- Keyboard navigation: logical Tab order, all interactive elements focusable

## VISUAL HIERARCHY RULES

**Primary Hierarchy (The Story)**

- One clear narrative thread per section. Secondary elements support, don't compete.
- Establish a visual pyramid: hero > supporting > detail > subtle
- Use size, color, and position to create focal points

**Color Hierarchy**

- **Dominant color (70%)**: Usually neutral background or light surface
- **Secondary color (20%)**: Brand accent, used for key messages and CTAs
- **Accent color (10%)**: Emotional warmth, signature moments, typography variety

**Size Hierarchy**

- H1: 80-200px (hero scale, breaks viewport, demands attention)
- H2: 48-80px (section anchors, major content divisions)
- H3: 32-48px (subsection titles, content grouping)
- Body: 16-18px (readable, generous line-height)
- Caption: 12-14px (metadata, supporting context)

**Cognitive Load Rule**

- No more than 3 distinct visual elements competing for attention per viewport
- Every additional element should support or clarify, not distract
- Use whitespace as hierarchy: generous negative space elevates content more than styling

**Whitespace as Hierarchy**

- Large sections of empty space communicate: premium, luxury, sophistication
- Dense sections require breathing room (generous margins/padding)
- Never fear whitespace—it's a design tool, not wasted space

## TYPOGRAPHY SYSTEM

**Type Pairing Philosophy**

- Display font: Bold, geometric, 1-2 typefaces maximum across entire site
- Body font: Neutral sans-serif, optimized for screen, designed for comfortable reading
- Accent font: Hand-lettered or decorative for emotional moments ONLY (use sparingly)

**Scale System** (Create Proportional Steps)

- Maintain consistent ratio between sizes (e.g., 1.125 or 1.25 multiplier)
- Create 6-8 defined sizes and reuse religiously—no arbitrary font sizes
- Example scale: 12px → 14px → 16px → 18px → 24px → 32px → 48px → 64px
- Document every size with use case: hero, h2, h3, body, caption, overline, etc.

**Spacing Rules**

- **Line-height**: 1.4 for headings (tighter), 1.6-1.8 for body text (open, readable), 1.2 for captions (compact)
- **Paragraph margins**: Consistent, usually 1.5-2x line-height
- **Letter-spacing**: Add slightly more to all-caps text (0.05-0.15em) and very large headings
- **Word-spacing**: Avoid widows/orphans through CSS `word-spacing` or careful copy editing
- **Line-length**: Aim for 45-75 characters (8-12 words per line) for body text comfort

**Color & Contrast**

- Minimum 4.5:1 contrast for body text on background (WCAG AA)
- 7:1 for critical text: form labels, CTAs, interactive elements
- Test color combinations with WebAIM Contrast Checker
- Never rely on color alone for meaning—always pair with text or icons

**Responsive Typography**

- Use fluid typography: CSS `clamp()` for smooth scaling between breakpoints
- Minimum: 14px body on mobile → 16px on desktop
- Preserve hierarchy ratios across breakpoints (don't crush large text on mobile)
- Test line-length at each breakpoint: ensure readability throughout
- Example: `font-size: clamp(1rem, 2vw, 1.5rem)` smoothly scales with viewport

**Text Treatment**

- Use semi-bold or bold for emphasis, not italics (clearer on screen)
- Avoid all-caps for body text (harder to read, slower scanning)
- Use all-caps strategically for labels, section markers, headlines
- Underlines reserved for links (never use for emphasis)

## MOTION GUIDELINES

### DO's

- ✓ Use motion to guide attention to key moments (hero reveals, section introductions)
- ✓ Time animations with natural curves (ease-out for exits, ease-in-out for smooth transitions)
- ✓ Pair motion with narrative meaning: content slides from left as it enters, fades in over background
- ✓ Always provide reduced-motion alternatives (CSS `prefers-reduced-motion: reduce` support)
- ✓ Prototype motion early: create video/GIFs before implementation
- ✓ Use consistent easing across site: pick 2-3 curves and reuse religiously
- ✓ Make animations intentional: every motion should serve a purpose (guide attention, provide feedback, create delight)

### DON'Ts

- ✗ Don't animate purely decorative elements with no semantic meaning
- ✗ Don't animate faster than 200ms (feels cheap, janky)
- ✗ Don't animate slower than 1.5s (feels sluggish, testing patience)
- ✗ Don't use bouncy easing on professional sites (spring easing feels playful/casual, not serious)
- ✗ Don't auto-play audio/video without explicit user permission
- ✗ Don't animate on every scroll (causes eye fatigue and motion sickness)
- ✗ Don't mix competing animation timings on same section (visual noise, disorienting)
- ✗ Don't animate properties other than transform/opacity on repeated animations (CPU-intensive, drops frames)

### ANIMATION CATEGORIES & USAGE

**1. Entrance Animations** (Page Load, Section Scroll-In)

- Patterns:
  - Fade + Scale: opacity 0→1, transform scale(0.95)→scale(1)
  - Fade + Slide: opacity 0→1, transform translateY(20px)→translateY(0)
  - Stagger on multiple elements: 50-100ms delay between each element
- Timing: 0.6-0.8s duration with cubic-bezier(0.34, 1.56, 0.64, 1) spring easing
- Use case: Hero headlines, section introductions, content block reveals
- Implementation: Intersection Observer or CSS animations triggered on scroll

**2. Scroll-Driven Animations** (Parallax, Reveal, Mask Animations)

- Parallax depth: Background moves slower than foreground (ratio: 0.5x scroll speed)
- Geometric mask reveals: SVG masks animated 0-100% over viewport height
- Text parallax: Large text moves faster to create cinematic effect
- Image parallax: Subtle movement creates depth illusion (use sparingly)
- Timing: Tied to scroll position (not time-based), implemented with ScrollTrigger or Intersection Observer
- Use case: Hero sections, portrait reveals, section dividers, full-page experiences
- Performance: Test frame rate—should maintain 60fps on desktop, 30fps on mobile

**3. Micro-Interactions** (Hover, Click, Focus)

- Button hover: Slight scale (1→1.05) + opacity shift (1→0.8) or color change
- Link underline: Grows from left to right on hover
- Form focus: Border color change, subtle glow, or shadow enhancement
- Toggle states: Icon rotates, color shifts, or background changes
- Checkbox/radio: Scale up slightly with color feedback
- Timing: 200-400ms with cubic-bezier(0.4, 0, 0.2, 1) ease-in-out
- Use case: ALL interactive elements—buttons, links, form fields, toggles
- Implementation: CSS transitions for simple states, JavaScript for complex interactions

**4. Transition Animations** (Page Changes, Modal Opens, Dropdown Expands)

- Fade transition: Opacity 0→1 over 0.4s
- Slide transition: New content slides in from direction (top/left/right)
- Scale transition: Modal opens from center with scale 0.8→1.0
- Timing: 0.4-0.6s with cubic-bezier(0.4, 0, 0.2, 1)
- Use case: Navigation changes, modal opens/closes, dropdown menus, tab switches
- Implementation: CSS transitions or JavaScript animation library

**5. Scene-Based Animations** (Complex Choreographed Sequences)

- Multiple elements animating in sequence with parent timeline (GSAP Timeline or Framer Motion orchestration)
- Staggered word reveals: text appears with slight scale and fade, words enter sequentially
- Sequential color shifts: text changes color as other elements move
- Coordinated movement: text and images move together with precise timing
- Timing: 1-2s total with internally coordinated stagger (100-200ms between elements)
- Use case: Hero sections, testimonial reveals, case study introductions, major page transitions

### TIMING & EASING REFERENCE

```css
/* Recommended Easing Functions */
--ease-standard: cubic-bezier(0.4, 0, 0.2, 1); /* iOS standard, professional */
--ease-in-out: cubic-bezier(
  0.4,
  0,
  0.6,
  1
); /* Smooth acceleration/deceleration */
--ease-out: cubic-bezier(0.34, 1.56, 0.64, 1); /* Spring-like, energetic */
--ease-out-cubic: cubic-bezier(
  0.215,
  0.61,
  0.355,
  1
); /* Professional, confident */
--ease-in-back: cubic-bezier(
  0.6,
  -0.28,
  0.735,
  0.045
); /* Anticipatory, entrance effect */
--ease-out-back: cubic-bezier(
  0.175,
  0.885,
  0.32,
  1.275

/* Timing Ranges */

--duration-long: 1200-1800ms; /* Full-page hero reveals, cinematic opens */
```

### PERFORMANCE CONSTRAINTS

- **Transform Only**: Animate `transform` and `opacity` ONLY for GPU acceleration (no width, height, position, padding, margins)
- **will-change**: Apply to animated elements only: `will-change: transform` (use sparingly, has performance cost)
- **Avoid Expensive Properties**: Never animate shadows, blur filters, gradients, border-radius (CPU-intensive, tanks performance)
- **Hardware Acceleration**: Test animations on real mobile devices—target 60fps on desktop, 30fps on mobile acceptable
- **Deferred Animations**: Lazy-load non-critical animations until after page is interactive (measure with Lighthouse Interactive metric)
- **Reduced Motion**: Implement `prefers-reduced-motion: reduce` for all animations—remove motion entirely for users with this preference
- **Performance Testing**: Use DevTools Performance tab to measure frame rate—maintain 60fps (16.7ms per frame) during animations

## 3D GUIDELINES

### When to Use 3D

- ✓ Create unique, memorable hero experiences (3D canvas, WebGL scenes)
- ✓ Stage products in realistic environments (AR/3D product demos)
- ✓ Create sophisticated parallax depth (multiple perspective layers)
- ✓ Differentiate brand in crowded competitive category
- ✗ NOT for decoration or novelty—always serves narrative or functional purpose

### How to Stage 3D

- **Hero Integration**: 3D model/scene occupies 40-60% of hero space; text remains 2D and readable
- **Focal Point**: Ensure 3D element draws attention without overwhelming primary message
- **Lighting**: Professional 3D uses realistic lighting (avoid overly saturated or artificial)
- **Fallback**: Always provide high-quality 2D image fallback for older browsers or low-end devices
- **Animation**: If 3D animates, use subtle rotations or camera pans (must maintain 60fps)

### Implementation Approach

- **Three.js/Babylon.js**: For complex scenes with full control
- **Spline**: Design tool with export capability—faster iteration for specific use cases
- **High-quality Video**: Alternative: render 3D as video if real-time 3D unnecessary
- **Profile Early**: Measure memory and GPU usage on target devices (mobile especially)

### Optimization

- **Polygon Reduction**: Reduce polygon count and texture resolution for performance
- **Level of Detail (LOD)**: High-quality models for desktop, simplified for mobile
- **Lazy Loading**: 3D loads only when in viewport
- **File Compression**: Use GLTF/GLB format with Draco compression
- **Performance Target**: 60fps on desktop, 30fps on mobile acceptable

### Accessibility & Fallbacks

- **Text Description**: Provide screen reader description of 3D scene content
- **Reduced Motion**: Disable 3D for `prefers-reduced-motion` users
- **Keyboard Navigation**: Ensure 3D doesn't interfere with keyboard control
- **Image Fallback**: High-quality 2D image for browsers without WebGL
- **Graceful Degradation**: Experience works fully without 3D (3D is enhancement, not requirement)

## RESPONSIVE STRATEGY

### Mobile-First Approach (Recommended)

1. Design base experience for mobile (320px+)
2. Add breakpoints and enhancements as screen size increases
3. Progressive enhancement: mobile works fully, desktop adds visual sophistication

### BREAKPOINT STRATEGY

```css
/* Standard Responsive Breakpoints */
$breakpoint-small: 480px; /* Small phones (iPhone 5, etc) */
$breakpoint-medium: 768px; /* Tablets (iPad, etc) */
$breakpoint-large: 1024px; /* Large tablets, small desktops */
$breakpoint-xlarge: 1200px; /* Desktop standard */
$breakpoint-2xl: 1600px; /* Large desktop, UHD monitors */
```

### LAYOUT REFLOW RULES

- **Mobile (<768px)**: Single column, full-bleed sections, hamburger navigation, stacked CTAs
- **Tablet (768-1024px)**: Two-column layouts possible, generous side margins (20-30px), sidebar navigation
- **Desktop (1024+)**: Full multi-column layouts, generous whitespace, max-width constraint (often 1200-1400px)
- **Ultra-wide (1600+)**: Consider max-width to maintain readable line-lengths, generous side margins

### COMPONENT BEHAVIOR CHANGES

- **Navigation**: Mobile hamburger menu → Tablet sidebar → Desktop horizontal bar
- **Grid Layouts**: 1 column (mobile) → 2 columns (tablet) → 3+ columns (desktop)
- **Typography**: Fluid scaling from smaller to larger via CSS `clamp()`
- **Images**: Full-width on mobile → Constrained width + side margins on desktop
- **Spacing**: Reduced padding/margins on mobile (constraint space), increased on desktop (luxury whitespace)
- **Modals**: Full-screen on mobile → Centered modal on desktop

### TOUCH VS. MOUSE INTERACTION

- **Touch targets**: Minimum 44-48px (not 32px) for comfortable tapping on mobile
- **Spacing**: Increase spacing between interactive elements on mobile (prevent misclicks)
- **Hover states**: Hidden on touch devices (no equivalent), visible on desktop
- **Scroll behavior**: Smooth scrolling on desktop, native momentum on mobile (faster, more native feel)
- **Form inputs**: Use native mobile inputs (date picker, number keyboard, color picker) for UX benefit

### PERFORMANCE ON MOBILE

- Reduce animation complexity: fewer parallax layers, simpler entrance animations
- Lazy-load images below fold with `loading="lazy"` attribute
- Defer non-critical JavaScript until after interaction (code splitting)
- Use modern image formats with fallbacks: WebP → JPEG, AVIF → WebP → JPEG
- Test on real 4G connection: Chrome DevTools throttling → Network: Fast 4G or Slow 4G

## STORYTELLING FRAMEWORK

### Three-Act Story Structure

**Act I: Exposition** (First Viewport, 0-300px scroll)

- Establish protagonist identity and current state
- Create curiosity through visual intrigue: fragmented text, mystery imagery, incomplete information
- Answer: "Who is this person/company and what do they do?"
- Visual strategy: Bold typography dominates, portrait photography creates personality
- Emotional tone: Interest, respect, curiosity, intrigue
- CTA: Implied (scroll to discover more)

**Act II: Inflection Point** (Mid-page, 300-1500px scroll)

- Introduce conflict, transformation, or shift (e.g., "After 20 years designing for clients... my mission changed")
- Show evidence of change: timeline, testimonials, work samples, client logos
- Answer: "Why should I believe this person/company can help me?"
- Visual strategy: Proof points (logos, numbers, credentials), candid team photos, diverse case examples
- Emotional tone: Trust, confidence, alignment with values
- CTA: Learn more about services or approach

**Act III: Resolution** (Final viewport, 1500px to bottom)

- Reveal new world, mission, or opportunity positioned for viewer as beneficiary
- Position viewer as protagonist in next chapter (they can achieve these goals too)
- Answer: "What's the next step? How do I engage?"
- Visual strategy: Call-to-action hierarchy, contact options, limited choices (reduce decision fatigue)
- Emotional tone: Inspiration, possibility, agency, excitement
- CTA: Clear primary action (schedule call, apply, get started) with secondary options

### COPY STRATEGY PRINCIPLES

**Lead with Transformation, Not Features**

- Instead: "I help design leaders think differently and challenge the status quo" (outcome-focused)
- Avoid: "I offer 1-on-1 coaching sessions with weekly check-ins" (feature-focused)
- Why: Prospects buy outcomes and transformation, not features

**Use Active Voice and Power Verbs**

- "Disrupt," "Challenge," "Create," "Innovate," "Transform," "Lead"
- Avoid passive voice: "designs can be improved" → "We'll elevate your designs"
- Creates agency, confidence, directness in tone

**Show, Don't Tell**

- Use imagery, testimonials, and examples instead of claims
- "20+ years with Fortune 500s and game-changing startups" (proof) > "I'm very experienced" (claim)
- "Students working at Apple, Google, Meta, IBM" (specific evidence) > "Helps students get jobs" (vague claim)

**Create Narrative Tension**

- Pose implicit questions, then answer them
- "Why would someone leave successful client work?" → "Because the impact of mentoring equals decades of billable hours"
- Questions create engagement—answers provide relief and clarity

**Emotional Pacing**

- Alternate between intellectual (facts, credentials, timeline) and emotional (human moments, possibilities, transformation)
- Balance: 40% proof/credentials, 40% human/emotional, 20% inspiration/future

**Specificity Over Generality**

- "Helped 500+ designers level up their leadership and land roles at tech leaders" (specific) > "Helps designers improve" (vague)
- "20 years at Discovery, Porsche, Edward Jones, American Express, AARP, ESPN" (named clients) > "Worked with major brands" (anonymous)
- Numbers create credibility: "Designed 150+ brand experiences" > "Extensive experience"

**Conversational Tone**

- Use "you," "let's," "I," "we"—direct address creates intimacy and breaks fourth wall
- "You might be wondering..." invites reader into conversation
- Contractions ("don't," "won't," "can't") make tone warm, not clinical

**Address Implicit Objections**

- "Not just talks—we dive deep with real examples from Fortune 500s and startups"
- "No hand-holding; I'll push you hard because you deserve excellence"
- Overcomes skepticism by acknowledging it directly

- **Icon System**: SVG sprites (inline or external) or icon fonts, optimized with SVGO
- **Optimization Tools**: ImageOptim for images, TinyPNG/TinyJPG for compression, uglify-js for scripts, cssnano for CSS

## DESIGN LEADERSHIP PRINCIPLES

### Craft Obsession

- Every pixel matters. Spacing is intentional, not arbitrary.
- Design system thinking: reusable patterns, consistent rules, systematic approach.
- Polish comes from discipline and attention to detail across entire experience.

### User-Centered Narrative

- Design solves problems; it doesn't decorate.
- Every visual decision supports the narrative and user journey.
- Audience insight informs hierarchy, pacing, and emotional tone.

### Performance as Design

- Fast, lightweight experiences are better UX than feature-rich, slow ones.
- Mobile performance shapes entire approach (mobile-first philosophy).
- Performance budgets are design constraints, not technical limits.

### Accessibility as Baseline

- Accessibility is non-negotiable—it's how you respect all users.
- Semantic HTML, keyboard navigation, and color contrast aren't "nice to have."
- Inclusive design creates better experiences for everyone.

### Motion with Purpose

- Motion should clarify, guide, or delight—never distract or overwhelm.
- Restraint in animation is sophisticated; excess is amateur.
- Respect users with `prefers-reduced-motion` preference.

### Systems Thinking

- Design components, not pages.
- Create rules and patterns; don't solve the same problem twice.
- Scalability matters: design system must support future growth.
